### .env

PORT = - порт текущего приложения например 3030
DATABASE_URL= - путь до базы данных например mongodb://localhost:27056/hotel_reservaton
SECRET= -текстовое значение для шифрования пароля
UPLOAD_LOCATION= место хранения загруженных фотографий комнат например './uploads/images'
MAX_FILE_SIZE = максимальный размер файла фото (байт)
SEPARATOR = число показывающее какая часть токена отправляется через Cookie
ADMIN_EMAIL= электронная почта администратора (при первой загрузке)
ADMIN_PASSWORD= пароль администратора не менее 6 символов (при первой загрузке)

## 1 Модуль Авторизации и Аутентификации

Авторизация и отправка в Cookie часть от JWT токена
часть определяется константой SEPARATOR, если равно 0, то авторизация и аутентификация только по JWT токену Bearer
токен с зашифрованным id пользователя
После выхода cookies убираются и пользователь становится не аутентифицирован.

При первой загрузке необходимо ввести в переменные окружения e-mail и пароль администратора (пароль должен быть больше или равен 6 символам)

## 2. Модуль гостиницы.

Функционал:

- создание администратором нового отеля;
- запрос списка гостиниц администратором;
- запрос администратором списка комнат в определенном отеле;
- обновление администратором информации об отеле;
- создание администратором новой комнаты для отеля;
- запрос информации любым пользователем о наличии свободных комнат;
- запрос информации любым пользователем об определенной комнате;
- редактирование администратором комнаты;

  ## Модуль навигация

  - навигация в зависимости от типа регистрации;

  ## Модуль резервирования комнаты в отеле

  Функционал:

  - резервирование клиентом комнаты;
  - получение клиентом списка зарегистрированных комнат;
  - удаление клиентом ранее рарезервированной комнаты;
  - получение менеджером списка комнат, зарезервированных пользователем;
  - удаление менеджером зарезервированных клиентом комнат;

## Модуль пользователей

Функционал:

- создание администратором нового пользователя;
- получение администратором и менеджером списка зарегистрированных клиентов;

## Модуль чат поддержки

Функционал:

- создание пользователем нового запроса в техподдержку;
- запрос пользователем списка чатов с техподдержкой;
- получение менеджером и клиентом списка сообщений в чате;
- отправка клиентом и менеджером сообщений в чат;
- отметка сообщений прочитанным менеджером и клиентом;
- запрос менеджером списка чатов;
